# Clash代理端口查找与配置指南

## 问题背景

在使用 mediaorganizer 下载海报时，配置了代理但仍然报错：

```
error sending request for url: error trying to connect: unsuccessful tunnel
```

配置文件中的代理设置：
```toml
[network]
proxy_enabled = true
proxy = "http://127.0.0.1:33331"
```

---

## 快速查找端口命令

### 方法一：查看配置文件（最可靠）

```bash
# 直接查看 Clash 配置文件中的端口设置
cat ~/.local/share/io.github.clash-verge-rev.clash-verge-rev/clash-verge.yaml | grep -E '^(mixed-port|port|socks-port):'

# 输出示例：
# mixed-port: 7897
```

### 方法二：从进程参数中提取配置文件路径

```bash
# 查找 verge-mihomo 进程，获取配置文件路径
ps aux | grep verge-mihomo | grep -oP '\-d\s+\K[^\s]+'

# 输出示例：
# /home/johnny/.local/share/io.github.clash-verge-rev.clash-verge-rev

# 然后查看该目录下的配置文件
CONFIG_DIR=$(ps aux | grep verge-mihomo | grep -oP '\-d\s+\K[^\s]+')
cat "$CONFIG_DIR/clash-verge.yaml" | grep -E '^(mixed-port|port|socks-port):'
```

### 方法三：查看所有监听端口并筛选

```bash
# 使用 ss 命令（推荐）
sudo ss -tlnp | grep -E 'mihomo|verge'

# 或使用 netstat 命令
sudo netstat -tlnp | grep -E 'mihomo|verge'

# 输出示例：
# tcp  LISTEN  0  4096  127.0.0.1:7897  0.0.0.0:*  users:(("verge-mihomo",pid=35474,fd=8))
```

### 方法四：一键查找并验证

```bash
# 提取端口并测试
PORT=$(cat ~/.local/share/io.github.clash-verge-rev.clash-verge-rev/clash-verge.yaml | grep '^mixed-port:' | awk '{print $2}')
echo "代理端口: $PORT"

# 测试 HTTP 代理
curl -x "http://127.0.0.1:$PORT" -I https://www.google.com --connect-timeout 5 && echo "✅ HTTP 代理可用"

# 测试 SOCKS5 代理
curl -x "socks5://127.0.0.1:$PORT" -I https://www.google.com --connect-timeout 5 && echo "✅ SOCKS5 代理可用"
```

### 方法五：使用 lsof 查找

```bash
# 查找 mihomo 进程打开的所有网络端口
sudo lsof -i -P -n | grep mihomo

# 只查看监听端口
sudo lsof -i -P -n | grep -E 'mihomo.*LISTEN'

# 输出示例：
# verge-mih  35474  johnny  8u  IPv4  123456  0t0  TCP 127.0.0.1:7897 (LISTEN)
```

## 问题诊断过程

### 第一步：初步查找 Clash 进程和端口

```bash
# 查找 Clash 相关进程
ps aux | grep clash

# 输出示例：
# johnny  35334  ... /usr/bin/clash-verge
# johnny  35474  ... /usr/bin/verge-mihomo -d /home/johnny/.local/share/...

# 查找监听端口
sudo netstat -tlnp | grep clash

# 输出示例：
# tcp  0  0 127.0.0.1:33331  0.0.0.0:*  LISTEN  35334/clash-verge
```

**初步结论**：看到端口 33331，但这实际上是 **Clash Verge GUI 的内部通信端口**，不是代理端口！

---

### 第二步：查看 Clash 配置文件（关键步骤）

根据进程输出，找到配置文件路径：

```bash
# 配置文件路径从进程参数中获取
cat ~/.local/share/io.github.clash-verge-rev.clash-verge-rev/clash-verge.yaml | head -20
```

**输出结果**：
```yaml
# Generated by Clash Verge

mode: rule
mixed-port: 7897          # ← 这才是真正的代理端口！
allow-lan: false
log-level: info
ipv6: true
external-controller: ''
secret: set-your-secret
unified-delay: true
...
```

**关键发现**：`mixed-port: 7897` 才是真正的代理端口！

---

### 第三步：验证端口是否可用

#### 3.1 测试错误的端口（33331）

```bash
# 测试 HTTP 代理
curl -x http://127.0.0.1:33331 -I https://www.google.com --connect-timeout 10

# 输出：
# HTTP/1.1 404 Not Found
# curl: (56) CONNECT tunnel failed, response 404

# 测试 SOCKS5 代理
curl -x socks5://127.0.0.1:33331 -I https://www.google.com --connect-timeout 10

# 输出：
# curl: (97) Received invalid version in initial SOCKS5 response.
```

**结论**：33331 不是代理端口，两种协议都失败。

#### 3.2 测试正确的端口（7897）

```bash
# 测试 HTTP 代理
curl -x http://127.0.0.1:7897 -I https://www.google.com --connect-timeout 10

# 输出：
# HTTP/1.1 200 Connection established
# HTTP/2 200
# ...（成功！）

# 测试 SOCKS5 代理
curl -x socks5://127.0.0.1:7897 -I https://www.google.com --connect-timeout 10

# 输出：
# HTTP/2 200
# ...（成功！）
```

**结论**：7897 是正确的代理端口，HTTP 和 SOCKS5 都可用。

## 正确的配置方法

### 方法一：修改配置文件

编辑 `~/.config/mediaorganizer/config.toml`：

```toml
[network]
proxy_enabled = true
proxy = "http://127.0.0.1:7897"
```

或使用 SOCKS5 协议：

```toml
[network]
proxy_enabled = true
proxy = "socks5://127.0.0.1:7897"
```

### 方法二：在 Clash Verge 中确认端口

1. 打开 Clash Verge 图形界面
2. 点击「设置」
3. 查看「混合端口」设置
4. 确认端口号（本例中是 7897）

## 一键查找脚本

创建脚本 `find_clash_port.sh`：

```bash
#!/bin/bash
echo "=========================================="
echo "Clash 代理端口查找工具"
echo "=========================================="

# 1. 查找进程
echo -e "\n[1] Clash 相关进程："
ps aux | grep -E 'clash|verge|mihomo' | grep -v grep

# 2. 查找配置文件中的端口
echo -e "\n[2] 配置文件中的代理端口："
CONFIG_FILE="$HOME/.local/share/io.github.clash-verge-rev.clash-verge-rev/clash-verge.yaml"
if [ -f "$CONFIG_FILE" ]; then
    echo "配置文件: $CONFIG_FILE"
    grep -E '^(mixed-port|port|socks-port):' "$CONFIG_FILE"
else
    echo "配置文件不存在: $CONFIG_FILE"
fi

# 3. 查找监听端口
echo -e "\n[3] 监听端口："
if command -v ss &>/dev/null; then
    sudo ss -tlnp 2>/dev/null | grep -E 'clash|verge|mihomo'
fi
if command -v netstat &>/dev/null; then
    sudo netstat -tlnp 2>/dev/null | grep -E 'clash|verge|mihomo'
fi

# 4. 提取并测试代理端口
echo -e "\n[4] 代理端口测试："
MIXED_PORT=$(grep '^mixed-port:' "$CONFIG_FILE" 2>/dev/null | awk '{print $2}')
if [ -n "$MIXED_PORT" ]; then
    echo "检测到混合端口: $MIXED_PORT"
    
    # 测试 HTTP 代理
    if curl -x "http://127.0.0.1:$MIXED_PORT" -I https://www.google.com --connect-timeout 5 &>/dev/null; then
        echo "✅ HTTP 代理可用: http://127.0.0.1:$MIXED_PORT"
    else
        echo "❌ HTTP 代理不可用"
    fi
    
    # 测试 SOCKS5 代理
    if curl -x "socks5://127.0.0.1:$MIXED_PORT" -I https://www.google.com --connect-timeout 5 &>/dev/null; then
        echo "✅ SOCKS5 代理可用: socks5://127.0.0.1:$MIXED_PORT"
    else
        echo "❌ SOCKS5 代理不可用"
    fi
fi

echo -e "\n=========================================="
echo "建议配置:"
echo "  proxy = \"http://127.0.0.1:$MIXED_PORT\""
echo "或"
echo "  proxy = \"socks5://127.0.0.1:$MIXED_PORT\""
echo "=========================================="
```

使用方法：
```bash
chmod +x find_clash_port.sh
./find_clash_port.sh
```

## 常见端口说明

Clash 及其变体通常使用以下端口：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `mixed-port` | 混合端口（HTTP + SOCKS5） | 7890 或 7897 |
| `port` | HTTP 代理端口 | 7890 |
| `socks-port` | SOCKS5 代理端口 | 7891 |
| `redir-port` | 透明代理端口 | 7892 |
| `external-controller` | 外部控制器 API | 127.0.0.1:9090 |

## 问题总结

### 为什么 33331 不工作？

1. **端口类型混淆**：通过 `netstat` 看到的 33331 是 Clash Verge GUI 程序的内部通信端口
2. **不是代理端口**：这个端口用于 GUI 和核心进程之间的通信，不接受代理请求
3. **正确的位置**：代理端口在配置文件的 `mixed-port` 字段中定义

### 正确的查找方法

1. **首选**：查看 Clash 配置文件中的 `mixed-port` 字段
2. **次选**：在 Clash Verge GUI 中查看设置
3. **验证**：使用 `curl` 测试端口是否可用
